CRITICAL ISSUE #2: SSH Root Login Enabled with Password Auth
================================================================================

STATUS:     Misconfigured
SEVERITY:   CRITICAL
FILE:       /etc/ssh/sshd_config
ISSUE:      Root can login directly via SSH with password

VULNERABILITY:
  Line: PermitRootLogin yes
  Line: PasswordAuthentication yes

IMPACT:
  - Highest privilege account (root) directly exposed to SSH
  - Brute force attacks can target root password
  - No normal escalation path required
  - Attackers bypass sudo/su auditing
  - Violates all security standards (CIS, NIST, etc)

ATTACK VECTORS:
  1. Dictionary attacks on root password
  2. Common password attempts
  3. Captured credentials from network
  4. Credential stuffing from breaches

QUICK FIX:
  sudo sed -i 's/^PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
  sudo sed -i 's/^PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
  sudo systemctl restart ssh

WARNING:
  After fix: SSH requires key-based authentication ONLY
  Ensure valid SSH keys exist before applying!
  Test: ssh -i ~/.ssh/id_rsa localhost

VERIFICATION:
  sudo sshd -T | grep -E "PermitRootLogin|PasswordAuthentication"
  Should show:
    permitrootlogin no
    passwordauthentication no

            /etc/nginx/.htpasswd.backup (mode 0645 - WORLD READABLE)
