# SEO Keyword Monitoring Platform - Project Context

## Project Overview

A comprehensive SEO monitoring and keyword tracking platform that provides:
- Multi-client keyword tracking
- Real-time ranking monitoring
- Analytics and reporting dashboard
- SerpBear integration for SERP data
- Unified client management interface

## Tech Stack

### Frontend
- **React** - UI framework
- **shadcn/ui** - Component library
- **Tailwind CSS** - Styling

### Backend
- **Node.js** - Runtime
- **Express** - Web framework
- **Database** - PostgreSQL/SQLite (check config)

### Infrastructure
- **Docker** - Containerization
- **Docker Compose** - Orchestration
- **Playwright** - E2E testing

### Services
1. **Dashboard Server** (`dashboard-server.js`) - Main unified dashboard
2. **Keyword Service** (`keyword-service/`) - Keyword tracking microservice
3. **SerpBear** (`serpbear/`) - Third-party SERP tracking integration

## Quick Start Commands

### Development
```bash

# Stop all services

# Run migrations
```

### Testing
```bash

node test-dashboard.cjs

# Run E2E tests
```

### Docker
```bash
# Build and start
docker-compose -f docker-compose.dashboard.yml up --build

# View logs
docker-compose -f docker-compose.dashboard.yml logs -f

# Stop and remove
docker-compose -f docker-compose.dashboard.yml down
```

## Architecture Patterns

### API Design
- RESTful endpoints under `/api/v1/`
- Consistent error response format
- JWT authentication (if implemented)
- Request validation middleware

### Database
- Migration-based schema management
- Referential integrity enforced
- Indexed for performance
- See `run-migration.js` for schema

### Service Communication
- HTTP/REST between services
- Health check endpoints
- Retry logic for failures
- Proper error propagation

## Coding Standards

### JavaScript/Node.js
- Use `const`/`let`, never `var`
- Async/await over callbacks
- Proper error handling with try-catch
- No console.logs in production code

### React
- Functional components with hooks
- shadcn/ui for UI components
- Prop validation
- Proper useEffect dependencies

### API Endpoints
- Input validation on all endpoints
- Consistent response format
- Proper HTTP status codes
- Error logging
